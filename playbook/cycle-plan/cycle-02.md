project: wedding-photo-share
cycle: 02
goal: 複数画像アップロードの安定化と、短時間ポーリング付きスライドショーの最小縦通しを実現する

## Scope（Must）
1. 複数画像アップロードの実用化
- 参加者が複数画像をまとめて選択できる
- 画像アップロードキューを実装し、同時実行数を3件までに制御できる
- キュー内の各ファイルについて pending / uploading / completing / done / error を管理できる
- 各ファイルごとに進捗または状態表示ができる
- 一部ファイルが失敗しても、他ファイルのアップロード処理が継続する
- failed のファイルのみ個別リトライできる
- complete 成功した投稿は一覧へ即時反映される

2. 投稿一覧の継続更新
- 参加者画面で投稿一覧を短時間ポーリングで自動更新できる
- posts API の since を使って差分取得できる
- 既存一覧への追記またはマージで二重表示を防げる
- uploaded かつ visible のみを表示し続ける

3. スライドショーの最小縦切り
- /room/:roomId/slideshow 画面を実装する
- スライドショー画面は room の有効性を確認できる
- 投稿一覧を短時間ポーリングで取得し、新着投稿を反映できる
- 表示対象の postId 群に対して短命 viewUrl をまとめて取得できる
- 画像を created_at 昇順でフルスクリーン表示できる
- 一定間隔で次の画像へ切り替えできる
- 投稿が0件のときは空状態メッセージを表示できる

4. viewUrl 一括発行の導入
- POST /api/rooms/:roomId/posts/view-urls を実装する
- uploaded かつ visible の post のみ短命URLを発行する
- 取得対象 postId のバリデーションができる
- フロントは必要な件数だけまとめて取得し、期限切れ前に再取得できる

5. スライドショー設定の最小利用
- GET /api/rooms/:roomId/slideshow-settings を利用できる
- interval_seconds をスライド切替に反映できる
- show_nickname の設定をスライド上に反映できる
- order_mode は cycle-02 では asc のみ実運用対象とし、manual は未対応でよい

## Scope（Should）
- 投稿一覧で自分の投稿を視認しやすくする
- スライド切替時に最小限のフェード演出を入れる
- 参加者画面でアップロード残件数や処理中件数を表示する
- viewUrl の再取得タイミングを共通 hook または service に切り出す

## Out of Scope（このサイクルではやらない）
- 動画アップロード
- 動画再生付きスライドショー
- 管理者による非表示 / 削除
- 管理者による表示順 manual 制御UI
- ZIP ダウンロード
- 合言葉付きスライドショー共有の磨き込み
- Cron による TTL 削除の本実装
- Durable Objects / SSE / WebSocket
- 本番品質のアニメーション最適化

## 技術方針
- 画像アップロードは cycle-01 の direct upload フローを拡張し、複数ファイルをキューでさばく
- 同時アップロード数は3件固定で開始し、設定化は後回しにする
- 投稿一覧とスライドショーは 3〜5秒のポーリングで更新する
- スライドショー用URLは都度個別取得ではなく、必要件数を一括取得する
- order_mode は API で返してもよいが、cycle-02 は asc 表示を優先し、設計先行部分は無理に広げない

## 完了条件
- 参加者が複数画像を選択し、3件並列までで安定してアップロードできる
- 一部失敗があっても他ファイルは継続し、失敗分だけリトライできる
- 参加者画面で投稿一覧がポーリングにより自動更新される
- スライドショー画面で新着画像が一定間隔で表示される
- viewUrl 一括発行 API を使って表示が過度にカクつかない
- 投稿0件 / 期限切れ / URL再取得失敗の最低限のハンドリングがある
- build が通る
- 既存の cycle-01 ユースケースを壊していない

## 人手確認
- スマホ回線相当で複数画像アップロードが極端に詰まらないか
- 連続投稿時に一覧の重複表示や欠落が起きないか
- スライドショー画面で数分間放置しても署名URL期限切れによる表示崩れが起きにくいか
- 投稿0件から新着1件目が自然に表示へ遷移するか

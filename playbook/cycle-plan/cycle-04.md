# Cycle 04 Plan — Wedding Theme / 参加者画面リッチ化

project: wedding-photo-share
cycle: 04
goal: 管理者が設定したテーマ画像・メッセージを参加者画面に反映し、結婚式らしい体験に改善する。スライドショーは投稿写真を一面表示するシンプル構成を維持する。

---

## 背景

Cycle-01〜03でMVP要件は達成済み。

- ルーム作成
- 参加者アップロード
- 投稿一覧更新
- スライドショー
- 管理者による非表示・削除
- スライドショー設定

が稼働確認済み。

次の改善として、結婚式当日に使うアプリとしての雰囲気を強化する。
ただし、スライドショー画面は投稿写真を主役にするため、装飾を増やさず、写真一面表示を維持する。

---

## Scope（Must）

### 1. テーマ設定データの追加

管理者がルームごとに以下を設定できるようにする。

- タイトル
- メッセージ
- メインビジュアル画像
- 背景画像
- テーマカラー
- アニメーションモード
  - none
  - fade
  - float

### 2. 管理画面へのテーマ設定UI追加

/admin/:roomId にテーマ設定セクションを追加する。

- タイトル入力
- メッセージ入力
- メインビジュアル画像アップロード
- 背景画像アップロード
- テーマカラー選択または入力
- アニメーションモード選択
- 保存ボタン
- 現在の設定プレビュー

### 3. テーマ画像アップロード

既存のR2直アップロード設計を流用して、テーマ画像を保存できるようにする。

- APIサーバー本体を経由しない
- 署名付きURLでR2へ直接PUT
- アップロード後、theme_settingsにfile_keyを保存
- 画像のみ対応
- 動画は対象外

### 4. 参加者画面へのテーマ反映

/room/:roomId にテーマ設定を反映する。

- 背景画像表示
- メインビジュアル表示
- タイトル表示
- メッセージ表示
- テーマカラー反映
- 投稿UIを結婚式向けに改善
- 設定がない場合でも既存UIが壊れない

### 5. スライドショーはシンプル維持

/room/:roomId/slideshow は、投稿画像を一面表示する方針を維持する。

- 投稿写真を主役にする
- 背景画像・装飾画像は原則表示しない
- 必要な改善はフェード・余白・表示崩れ対策程度に留める

---

## Scope（Should）

- テーマ設定保存後、参加者画面で即反映しやすい構成にする
- 背景画像は読み込み失敗時にフォールバックする
- メインビジュアル未設定でも自然な表示にする
- スマホ表示を優先して整える
- 管理画面のテーマプレビューを簡易表示する

---

## Scope（Could）

- 背景画像に軽いぼかし・暗めオーバーレイを入れる
- メインビジュアルにゆっくり浮遊アニメーションを入れる
- ボタンやカードにガラス風表現を入れる
- テーマカラーから簡易的にボタン色を反映する

---

## Out of Scope

- スライドショーの過剰装飾
- スライドショーへのテーマ画像表示
- 動画背景
- 複数テーマ管理
- ドラッグ&ドロップ配置
- 高度なフォント設定
- AI画像生成
- アニメーションの大規模作り込み
- 管理者権限の高度化
- 参加者ログイン機能

## 技術方針

### DB

新規テーブル theme_settings を追加する。

主な項目：

- room_id
- title
- message
- main_visual_key
- background_image_key
- theme_color
- animation_mode
- updated_at

### API

Workersに以下を追加する。

- GET /api/rooms/:roomId/theme
- PUT /api/rooms/:roomId/theme
- POST /api/rooms/:roomId/theme/upload-url
- POST /api/rooms/:roomId/theme/view-urls

管理者更新系は X-Host-Token 必須。
参加者取得系はルーム有効期限とpasscode設定がある場合の検証を行う。

### Storage

R2の保存先例：

- {roomId}/theme/main-visual.{ext}
- {roomId}/theme/background.{ext}

### Frontend

- 管理画面にテーマ設定フォームを追加
- 参加者画面にテーマ表示コンポーネントを追加
- 既存アップロード体験は壊さない
- スライドショーは基本既存維持

# Cycle-06 Implementation Tasks

## ゴール
本番デプロイ前に、既存機能を大きく広げず、結婚式当日に使えるレベルへUXと安定性を整える。
デプロイ作業と本番チェックリスト作成は行わない。

---

## 1. 作業開始前確認

- main の最新を pull する
- 必要なら作業ブランチを切る
- build が通る現状を確認する
- Cycle-01〜05 の主要導線をざっと確認する

主要導線：
- ルーム作成
- 参加者入室
- 画像アップロード
- ギャラリー表示
- 画像保存
- 管理者ログイン
- 投稿非表示/削除
- テーマ設定

---

## 2. 共通UI状態の整理

### 2-1. Loading 表示
以下にローディング状態を追加または整理する。

- ルーム取得中
- テーマ取得中
- 投稿一覧取得中
- viewUrl取得中
- 管理画面データ取得中
- ダウンロード処理中

### 2-2. Empty 表示
以下の空状態を明示する。

- 投稿がまだない
- ギャラリー画像がない
- 選択中の画像がない
- 未保存画像がない
- 管理画面で投稿がない

### 2-3. Error 表示
共通方針：
- 技術的すぎる文言を避ける
- 次に何をすればよいか分かる文言にする
- 画面全体を壊さない

---

## 3. 参加者画面のスマホUX改善

### 3-1. レイアウト調整
- スマホ幅で横スクロールが出ないようにする
- ボタンのタップ領域を十分にする
- 主要操作を見つけやすくする
- テーマ画像がある場合でも文字が読めるようにする

### 3-2. 文言調整
- 投稿導線の文言を自然にする
- ギャラリー導線の文言を自然にする
- エラー時の文言を短く分かりやすくする

---

## 4. アップロード安定化

### 4-1. 状態表示
各ファイルで以下の状態が分かるようにする。

- 待機中
- アップロード中
- 登録中
- 完了
- 失敗

### 4-2. リトライ
- 失敗したファイルだけリトライできる
- リトライ時に状態が正しく戻る
- 他のファイルに影響しない

### 4-3. 例外処理確認
以下で try/catch があり、fail API または適切な失敗状態へ遷移することを確認する。

- upload-url取得失敗
- R2 PUT失敗
- complete失敗

### 4-4. 連打・二重実行対策
- アップロード中に同じファイル処理が重複実行されないようにする
- 処理中ボタンは disabled などで制御する

---

## 5. ギャラリー保存UX改善

### 5-1. 選択状態
- 選択数が分かる
- 全選択/全解除の挙動が直感的
- 保存済み画像が視覚的に分かる

### 5-2. ダウンロード進捗
- 現在何件中何件目か表示する
- 完了件数、失敗件数が分かる
- 一部失敗しても処理を継続する

### 5-3. localStorage耐性
- savedPostIds の値が壊れていても落ちない
- 空配列として復旧できる

### 5-4. ダウンロードボタン制御
- 対象0件のときは disabled
- 処理中は多重実行できない

---

## 6. 管理者画面の運用性改善

### 6-1. 認証エラー
- host_token 未入力時の案内
- host_token 不一致時の案内
- 再入力しやすいUI

### 6-2. 投稿管理
- hidden / visible の状態が分かりやすい
- 削除前確認を明確にする
- 削除成功/失敗の表示を出す

### 6-3. テーマ設定
- 保存中状態を表示
- 保存成功/失敗を表示
- 画像アップロード中状態を表示
- 未設定状態でも崩れない

---

## 7. スライドショーの現状維持確認

スライドショーは大きく変更しない。
確認のみ行う。

- フルスクリーン画像表示が壊れていない
- hidden画像が表示されない
- 設定したintervalが反映される
- 画像0件時に真っ白にならない

---

## 8. APIエラー応答の最低限整理

必要に応じて、既存APIのエラーレスポンスを最小限整理する。

対象：
- 401 / 403
- 404
- 期限切れルーム
- バリデーションエラー
- R2 / D1 操作失敗

方針：
- 大幅なAPI再設計はしない
- フロントが表示しやすい message を返す程度に留める

---

## 9. 制約

- デプロイしない
- 本番チェックリストは作らない
- 新機能を広げない
- 動画対応しない
- ZIP生成しない
- 不要な依存を追加しない
- 既存構成を壊さない

---

## 10. 完了条件

- build が通る
- 参加者画面がスマホで大きく崩れない
- アップロード失敗時にリトライできる
- ギャラリー保存中の進捗が分かる
- 一部ダウンロード失敗でも全体が止まらない
- 管理者操作の成功/失敗が分かる
- 各画面のローディング/空/エラー状態が最低限表示される
- Cycle-01〜05 の主要機能が壊れていない

- 確認した導線
